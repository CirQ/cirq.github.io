An interesting plugin: hexo-blog-encrypt

发表于 2020-01-04 更新于 2020-02-11 分类于 techniques

Overview

You find the content! Try it!

Story

在hexo的插件列表里面找到一个有意思的插件，用来加密文章的。一开始想的是，解密的所有操作都发生在前端，怎么能够保证隐私性呢？于是去看了源码，发现自己在应用密码学上的思路还是欠缺了，所以在这里多写一段关于这个插件是怎么工作的。

在要加密的文章头部加一个password:字段，用来声明加密的密码，实际上也就是用来加密的密钥。加密的算法用到了两个部件，一个是加密算法AES256-CBC，另一个是消息认证码HMAC-SHA256。密钥是通过PBKDF2算法产生的，可以避免基于爆破密钥的攻击。当然，在这个场景下爆破攻击仍然存在，但通过这种密钥推导（key derivation）也能够生成较为安全的密钥。

具体的加密算法接受两个参数(pw, msg)，代表密码和文章，输出也是两元组(c, h)，密文和一个消息摘要。加密的过程如下：

可以看到，key和iv都是同一个密码推导出来的，但这不算威胁，因为本来IV就被设计成可以公开的密文的一部分（另一个角度来看，iv的存在暴露了和key的联系。不过由于SHA256是单向的，这不是大问题）。这样来看，剩下的威胁是三个：

  • IV是确定性算法生成的，相当于整个加密系统的随机性都来源于密钥（实际上是密码）
  • 加密算法和消息认证码采用了相同的密钥，相当于hmac没起到认证的作用
  • 用E&A的认证模式，本质上也是不安全的

说来也有趣，后两点都在这学期修的CS403中讲到了。但它们都是在理论层面上声称为不安全的。实际上，作者加入hmac的目的也不是为了认证，而是为了在解密成功时，避免错误的明文造成困扰（padding-oracle的存在告诉我们，只有当填充不正确时，才会出现解密失败）。通过上述分析，基本可以认为，这个系统的弱点只可能来源于IV。但现在还不能证明这个加密是（不）安全的，所以就到此为止吧【烂尾】。
