现在是大年初六，距离发表应该还有一段时间，真希望这个被我坑了很久的课程能尽快完成。

Lec.13
Content: Dynamic programming: overlapping subproblems, optimal substructure
Link: here

求斐波那契数列应该是最简单的动态规划的应用，如果用递归解法，会产生大量的overlapping subproblems。如果不做记忆化，那么重复子问题会被多次求解，时间复杂度也就成指数倍增长。

定义相互重叠的子问题，并对子问题做记忆化，就是动态规划(dynamic programming)的核心。这与分治(divide and conquer)算法是很不一样的，尽管两者的子问题都要求结构相同，但分治的子问题之间是相互独立的。当用于解决优化问题时，DP要求每个子问题有最优的子结构，这样，最终的结果才是最优的。

动态规划可以很有效地解决01背包问题，课上给出的方案就是普通的回溯加上记忆化，效率也能够得到极大的提升。

Lec.15
Content: Abstract data types, classes and methods
Link: here

模块化(modularization)的目的是将具有相关联的函数与数据进行打包，而面向对象编程中的类(class)就是一种特殊的模块。它的特殊之处在于可以实例化，这样各个实例可以封装自身的私有数据，共享公有的数据和方法等。

剩下的便是一些简单的python的面向对象特性语法。python提供了一个dir([object])函数，可以方便地查看对象所有的属性和方法，可以通过重写对象的__dir__方法来覆盖它的行为。

另外，python自带的help函数能用于查看任何对象、模块的文档，十分便利。

Lec.16
Content: Encapsulation, inheritance, shadowing
Link: here

老生常谈的面向对象三大特性：封装(encapsulation)、继承(inheritance)、多态(polymorphism)，最后一个也称作shadowing。

现代一点的高级语言基本都支持面向对象的编程范式，或多或少地实现了这些特性，详细讨论起来肯定是一片大文章。考虑到本人水平有限，也就无法展开了。然而我至今还是认为，面向对象带来的讨论，特别是设计模式，是某些程序语言社区的阴谋，为了让人们的重心脱离编程本身，陷入解决实际问题以外的无底洞，从而树立起宗教一样的、以单一种（或一类）语言为中心的党派，割裂程序员群体。这可以说是相当无意义的了。

Lec.17
Content: Computational models: random walk simulation
Link: here

这一节开始，课程转向了计算问题(computational problem)。与精心设计的像动态规划或分治算法不同，这一类问题的一个特点是，使用大量的重复计算来逐步得到最终结果，而这通常难以用纸笔演算得到。这大概就是计算机的终极美学了。

这类计算模型通常类似于暴力(brute force)，且带有一定的随机性，目的就是通过大量的随机试验来取得接近实际问题的结果。其中一个最简单的模型就是随机游走(random walk)。

一个人站在二维网格的原点，每秒随机向上下左右的其中一个方向移动一单位距离。经过一段时间后，他离原点的期望距离有多远？

一个很显然的考虑是，这个距离是否会随着时间的增加而变长？简单地想，当步数为奇数时，是无法回到原点的；而当步数为偶数时，假设人走了两步，那他只有1/4的几率回到原来的位置。这样看，我们很难认定最终距离会是常数。这时候就可以发挥计算机程序的暴力美学了：通过模拟这一事件，多次试验后期望能接近一个合理的结果。

Lec.18
Content: Presenting simulation results, Pylab, plotting
Link: here

多次试验后，上节提出的问题的答案并不随着步数增加而收敛，也就是说，离原点的距离是随着时间一直增长的。课程上只给了仿真的结果，但实际上，最终距离的期望是$\sqrt{N}$，其中$N$是所走的步数，且与空间的维度无关，这个结论与课堂仿真的结果实相符的。通过谷歌可以查到许多关于这个结论的推导过程，这里只给出一种简化情况的证明。

考虑二维网格中，人每步前进距离为1，且可选择的有四个方向，那么一共会产生四种可能的单位向量：$(1, 0),\ (-1, 0),\ (0, 1),\ (0, -1)$。将这些向量标记为$(x_i, y_i)$，那么$N$步以后就会产生$N$个向量，它们的和就是最终偏移原点的结果。根据初等的几何原理，可以得到该距离的平方是：

显然$x_i^2+y_i^2=1$，由于$x_i$和$y_i$是均匀分布的，因此后面的混合积之和的期望值应该为0。于是我们得到结果：$R=\sqrt{n}$。

这个推导过程来源于《从一到无穷大》第八章，注意到这个结论不依赖于方向是否正交，甚至不限定在二维空间！在欧式空间中，只要每一步的距离都是1，那么最终的期望偏离原点的距离就是步数的平方根，这是一个相当优美的结论。
